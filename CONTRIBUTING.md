# راهنمای مشارکت

مشارکت عمومی در Dicode Config Checker فعلاً روی نگهداری و بهبود فهرست کانال‌های عمومی متمرکز است.

## پیشنهاد کانال جدید

برای پیشنهاد یک کانال، از فرم **Channel suggestion** در بخش Issues استفاده کنید. هر Issue فقط برای یک کانال ثبت شود تا بررسی و پیگیری آن ساده بماند.

کانال پیشنهادی باید این شرایط را داشته باشد:

- لینک عمومی و مستقیم با قالب `t.me/channel_username` داشته باشد؛
- محتوای اصلی آن انتشار کانفیگ یا پروکسی قابل استخراج باشد؛
- برای مشاهده اولیه به عضویت اجباری یا دسترسی خصوصی نیاز نداشته باشد؛
- در `channels.txt` موجود نباشد؛
- فعالیت قابل قبول و محتوای مرتبط داشته باشد؛
- عمدتاً اسپم، تبلیغات نامرتبط یا محتوای فریبنده منتشر نکند.

## گزارش کانال نامعتبر

اگر کانالی تغییر کاربری داده، حذف شده، خصوصی شده یا دیگر محتوای مرتبط منتشر نمی‌کند، همان فرم پیشنهاد کانال را باز کنید و نوع درخواست را روی «حذف یا اصلاح کانال موجود» قرار دهید.

## روند بررسی

1. لینک و عمومی بودن کانال بررسی می‌شود.
2. نوع محتوای منتشرشده و امکان استخراج لینک‌ها بررسی می‌شود.
3. تکراری نبودن نام کاربری در فهرست کنترل می‌شود.
4. پس از تایید، کانال به `channels.txt` اضافه یا اصلاح می‌شود.
5. حساب GitHub پیشنهاددهنده در سابقه Issue و در صورت مشارکت تاییدشده، در بخش مشارکت‌کنندگان کانال ثبت می‌شود.

پذیرش یک پیشنهاد به معنی تایید محتوای کانال یا تضمین دائمی کیفیت کانفیگ‌ها و پروکسی‌های آن نیست.

## تغییر مستقیم فهرست

Pull Request برای تغییر `channels.txt` نیز قابل قبول است. در این حالت:

- هر خط فقط یک لینک `t.me` داشته باشد؛
- ترتیب فعلی تا حد امکان حفظ شود؛
- کانال تکراری اضافه نشود؛
- در توضیح Pull Request دلیل اضافه، حذف یا اصلاح شدن کانال نوشته شود.

برای تغییرات خارج از فهرست کانال‌ها، ابتدا یک Issue باز کنید تا محدوده تغییر مشخص شود.
